**Atividade Avaliativa: Tela Inicial de um E-commerce com ReactJS e  Consumo de API** 

**Objetivo** 

Desenvolver uma tela inicial para um e-commerce utilizando ReactJS, que consuma uma  das APIs externas disponíveis (https://fakeapi.platzi.com ou https://fakestoreapi.com). A  atividade avaliará a capacidade de implementar componentes reutilizáveis, consumir APIs  RESTful, e exibir os dados de forma organizada e funcional. 

***Descrição da Atividade*** 

1\. Escolha uma das APIs fornecidas (https://fakeapi.platzi.com ou  

https://fakestoreapi.com). 

2\. Consuma os dados de produtos fornecidos pela API. 

3\. Crie uma interface em ReactJS que exiba os seguintes elementos: a. Nome do produto 

b. Imagem do produto 

c. Preço do produto 

d. Categoria do produto 

4\. A interface deve permitir a exibição de pelo menos 10 produtos e ser responsiva  para diferentes tamanhos de tela. 

5\. A interface deve ser implementada utilizando o conceito de **componentização**. Por  exemplo: 

a. Um componente para a listagem dos produtos. 

b. Um componente individual para cada produto. 

***Requisitos Técnicos*** 

• Utilize ReactJS com **hooks** (useState e useEffect) para o consumo e manipulação  dos dados da API. 

• Implemente **estilização básica** utilizando CSS, Styled Components, ou bibliotecas  como Material-UI.  
• Adicione um carregamento condicional para exibir uma mensagem de  "Carregando..." enquanto os dados da API estão sendo buscados. 

• Adicione tratamento de erros para exibir uma mensagem caso a API não possa ser  acessada. 

***Critérios de Avaliação*** 

| Critério | Pontuação   Máxima |
| ----- | ----- |
| **Consumo da API**: Utilização de fetch ou axios para consumir os  dados da API escolhida. Tratamento correto da resposta da API. | **20 pontos** |
| **Componentização**: Criação de componentes reutilizáveis para  estruturar a aplicação (ex.: componentes de produto e listagem). | **20 pontos** |
| **Apresentação dos dados**: Exibição clara e organizada dos dados na  interface. Utilização de imagens, preços e categorias. | **20 pontos** |
| **Estilização**: Interface esteticamente agradável e responsiva,  adaptada a diferentes dispositivos. | **15 pontos** |
| **Tratamento de erros**: Implementação de mensagens de erro e  carregamento enquanto os dados estão sendo buscados. | **15 pontos** |
| **Código limpo e boas práticas**: Organização do código, uso de nomes  descritivos, e ausência de código desnecessário. | **10 pontos** |

***Entrega*** 

1\. Submeta o código em um repositório no GitHub ou em um arquivo zip enviado para  o classroom. 

2\. Inclua um arquivo README.md com instruções de como rodar a aplicação. 

**Total de Pontos: 100 pontos**
